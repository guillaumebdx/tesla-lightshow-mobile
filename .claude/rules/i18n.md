# Internationalization Rules

## File locations
- `src/i18n/en.json` — English (primary/reference)
- `src/i18n/fr.json` — French
- `src/i18n/de.json` — German
- `src/i18n/es.json` — Spanish

## Mandatory rule
Every new user-facing string MUST be added to ALL 4 locale files simultaneously. Missing keys will cause the app to show raw key names to users.

## Key structure
Keys are organized by section matching the screen/feature:
- `app` — global app strings
- `home` — HomeScreen (show list, settings, about)
- `settings` — settings modal
- `newShow` — NewShowScreen (model picker, creation)
- `editor` — ModelViewer (settings panel, actions)
- `aiPrompt` — AI generation modal
- `timeline` — AudioTimeline (track picker, playback)
- `parts` — PartOptionsPanel (vehicle part names, effects)
- `export` — ExportModal (tutorial pages)
- `share` — share modal
- `flash` — toast messages
- `tutorial` — first-time tutorial
- `load` — load modal
- `chat` — support chat

## Usage in code
```javascript
import { useTranslation } from 'react-i18next';
const { t } = useTranslation();
// Then: t('section.key') or t('section.key', { variable: value })
```

## Never hardcode user-facing text
Even short labels like "OK" or "Cancel" must go through `t()`.
